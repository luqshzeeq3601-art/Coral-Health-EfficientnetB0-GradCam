# Spec Review Checklist

Ask these questions while reviewing a spec for implementation and execution readiness:

1. Is the problem statement specific and grounded in current failures?
2. Are non-goals explicit?
3. Are contracts concrete enough to implement?
4. Are acceptance criteria testable?
5. Are execution rules fully defined (no reliance on implicit runtime assumptions)?
6. Does the spec define the source of truth for new fields, states, or behaviors?
7. Does it match existing code paths and public tool surfaces?
8. Can each open concern be classified as a spec issue, execution gap, or implementation issue?
9. Is the spec ready to execute and implement without further interpretation?
